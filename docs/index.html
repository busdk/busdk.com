Business Development Kit (BusDK)

A modular, Git-native, CLI-first toolkit for running a business — built on open, long-lived formats and transparent, auditable workflows, designed for humans and AI agents.

BusDK workspaces live in Git repositories. Business data is stored as UTF-8 CSV datasets validated with Frictionless Data Table Schemas (JSON). The system keeps operations transparent and auditable, with deterministic CLI workflows that serve both humans and AI agents.

Status: pre-release, under active development. Interfaces and schemas may still change.

Modules (subcommands)

BusDK is designed as focused binaries invoked through the core bus dispatcher.

  • bus-init — Bootstraps a new workspace by orchestrating module-owned init commands and creating the chosen workspace layout.
  • bus-config — workspace configuration tooling (accounting entity settings in datapackage.json).
  • bus-dev — developer workflows for module repositories (commit, work, spec, e2e).
  • bus-agent — diagnostics and development helper for the agent runner (detect, render, run, format).
  • bus-bfl — BusDK Formula Language tooling for expressions.
  • bus-data — Tabular data layer: schema-validated dataset I/O and validation for BusDK workspaces (CSV + JSON Table Schema), providing a Go library (and a thin bus data … CLI) for deterministic CRUD-style table and schema handling without domain business logic or CLI-to-CLI dependencies.
  • bus-api — Local REST JSON API gateway for BusDK workspaces.
  • bus-sheets — Local spreadsheet-like web UI for workspaces (workbook, sheet tabs, optional agent chat).
  • bus-accounts — chart of accounts / account master data.
  • bus-assets — fixed asset register and depreciation schedules.
  • bus-attachments — document archive with schema metadata.
  • bus-bank — bank statement import and reconciliation.
  • bus-budget — budgets and variance reporting.
  • bus-entities — durable IDs for customers, vendors, banks, and authorities.
  • bus-inventory — inventory and stock records with valuation support.
  • bus-invoices — sales and purchase invoices with validation.
  • bus-journal — double-entry journal and ledger validation.
  • bus-loans — loan register, drawdowns, and amortization schedules.
  • bus-payroll — payroll runs and journal posting outputs.
  • bus-period — close and lock accounting periods.
  • bus-reconcile — reconciliation state and payment allocation.
  • bus-reports — trial balance, ledger, and financial statements.
  • bus-validate — workspace validation and invariants.
  • bus-vat — Finnish VAT (ALV) computations and checks.
  • bus-pdf — Renders deterministic PDFs from prepared JSON render models without modifying any canonical workspace datasets.
  • bus-filing — authority-ready filing bundles.
  • bus-filing-prh — PRH filing target exporter.
  • bus-filing-vero — Vero filing target exporter.
