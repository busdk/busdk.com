Business Development Kit (BusDK)

A modular, Git-native, CLI-first toolkit for running a business — built on open, long-lived formats and transparent, auditable workflows, designed for humans and AI agents.

BusDK workspaces live in Git repositories. Business data is stored as UTF-8 CSV datasets validated with Frictionless Data Table Schemas (JSON). The system keeps operations transparent and auditable, with deterministic CLI workflows that serve both humans and AI agents.

Status: pre-release, under active development. Interfaces and schemas may still change.

Modules (subcommands)

BusDK is designed as focused binaries invoked through the core bus dispatcher.

  • bus init — Initialize a new workspace by orchestrating module-owned init commands.
  • bus config — Create and update workspace configuration (accounting entity settings in datapackage.json).
  • bus data — Inspect workspace datasets and schemas through a minimal data-layer CLI.
  • bus api — Local REST JSON API gateway for BusDK workspaces.
  • bus sheets — Local spreadsheet-like web UI for workspaces (workbook, sheet tabs, optional agent chat).
  • bus dev — Developer workflows for module repos (commit, work, spec, e2e); not for end-user bookkeeping.
  • bus agent — Diagnostics and development helper for the agent runner (detect, render, run, format).
  • bus bfl — Developer CLI for BusDK Formula Language expressions.
  • bus accounts — Manage the chart of accounts.
  • bus entities — Manage counterparty reference data.
  • bus period — Open, close, and lock accounting periods.
  • bus attachments — Register evidence files and attachment metadata.
  • bus invoices — Create and manage sales and purchase invoices.
  • bus journal — Post and query ledger journal entries.
  • bus bank — Import and list bank transactions.
  • bus reconcile — Match bank transactions to invoices or journal entries.
  • bus assets — Manage fixed-asset records and depreciation.
  • bus loans — Manage loans and amortization schedules.
  • bus inventory — Manage inventory items and movements.
  • bus payroll — Run payroll and generate postings.
  • bus budget — Record budgets and run budget vs actual reports.
  • bus reports — Generate trial balance, ledger, and statement reports.
  • bus validate — Validate workspace datasets and invariants.
  • bus vat — Compute VAT reports and exports.
  • bus pdf — Render PDFs from JSON render models.
  • bus filing — Build deterministic filing bundles.
  • bus filing prh — Produce PRH export bundles.
  • bus filing vero — Produce Vero export bundles.
